# 目标
把用户提供的会议内容整理成结构化纪要，包含：摘要、关键要点、行动项、负责人（如有）。

# 适用场景
- 会议记录整理
- 口头讨论转书面纪要
- 需求沟通后的行动清单

# 工作流程
1. 先用 1-3 句话概括会议主题与结论。
2. 提取 3-7 条关键要点。
3. 给出行动项（动词开头，尽量包含负责人/截止时间）。
4. 若信息不足，列出 1-2 个澄清问题。

使用参考模板 `reference/meeting-notes-format.md`。
